# 暗黑模式

## 功能说明

支持 `light`、`dark`、`system` 三种主题模式。

当前项目除了切换网页内容区的 CSS 变量外，还会在 **Windows** 下同步原生标题栏颜色，避免出现应用内容是浅色、窗口顶栏仍然发黑的情况。

## 相关文件

- `src/lib/use-theme.ts`：主题 Hook
- `src/index.css`：主题颜色变量
- `src-tauri/src/window_chrome.rs`：Windows 标题栏颜色同步
- `src-tauri/src/lib.rs`：注册 `sync_window_theme` 命令

## 实现原理

### 1. CSS 变量

`src/index.css` 中维护明暗两套 token：

```css
@theme {
  --color-background: #ffffff;
  --color-card: #ffffff;
}

.dark {
  --color-background: #171717;
  --color-card: #262626;
}
```

`useTheme()` 会在 `<html>` 上切换 `.dark` 类，Tailwind 与 CSS 变量共同驱动整体配色。

### 2. 跟随系统

当主题设置为 `system` 时，监听 `prefers-color-scheme`：

```ts
const mq = window.matchMedia('(prefers-color-scheme: dark)')
const handler = () => applyTheme('system')
mq.addEventListener('change', handler)
```

### 3. Windows 原生标题栏同步

`dev-tools` 中已经做过这层处理，`dev-env` 现在也同步了同样的能力。

`use-theme.ts` 在切换主题时会额外调用：

```ts
await invoke('sync_window_theme', { dark })
```

Rust 侧通过 `DwmSetWindowAttribute` 设置：

- `DWMWA_CAPTION_COLOR`
- `DWMWA_TEXT_COLOR`
- `DWMWA_BORDER_COLOR`

这样可以让标题栏和侧栏、卡片背景保持一致。

## 持久化

主题选择保存在 `localStorage`。

当前实际 key 仍是：

- `dev-tools-theme`
- 兼容旧 key：`tool-kit-theme`

这是历史迁移遗留，代码当前仍沿用该 key，文档按实际实现记录。

## 注意事项

- 在浏览器预览模式下，标题栏同步不会生效，因为没有 Tauri 原生窗口。
- 修改主题相关逻辑后，最好同时执行 `pnpm build` 和 `cargo check --manifest-path src-tauri/Cargo.toml`。

## 二次开发

- 修改颜色：编辑 `src/index.css`
- 扩展主题行为：编辑 `src/lib/use-theme.ts`
- 调整 Windows 标题栏颜色：编辑 `src-tauri/src/window_chrome.rs`
