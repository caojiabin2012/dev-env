# 关于与更新

## 功能说明

当前“关于 / 更新”相关能力主要集中在设置页的“关于”标签中，负责展示：

- 应用版本
- 更新检查入口
- 更新安装入口
- 功能分组说明

更新底层基于 `tauri-plugin-updater` 与 GitHub Release 生成的 `latest.json`。

## 相关文件

- `src/components/settings.tsx`：设置页中的“关于”标签
- `src/components/about.tsx`：保留的独立关于页
- `src/lib/settings-api.ts`：更新安装 API
- `src/lib/updater.ts`：前端更新检查封装
- `src/lib/use-update.ts`：启动时检查更新 Hook
- `src-tauri/src/settings.rs`：版本号读取与更新安装
- `src-tauri/tauri.conf.json`：updater endpoint 配置

## 当前实现状态

### 已接通

- Rust 侧 `install_update_and_restart` 已接入 `tauri-plugin-updater`
- Release Workflow 会生成 `latest.json`
- `tauri.conf.json` 已配置：

```json
"plugins": {
  "updater": {
    "endpoints": [
      "https://github.com/caojiabin2012/dev-env/releases/latest/download/latest.json"
    ]
  }
}
```

### 当前前端检查逻辑

`src/lib/updater.ts` 目前仍是一个占位实现：

```ts
export async function checkForUpdate() {
  await getVersion()
  return { status: 'up-to-date' }
}
```

也就是说：

- UI 中“检查更新”入口还在
- 但“是否有新版本”的前端查询逻辑尚未完全恢复
- 文档在这里按实际代码状态记录，不再按旧版 `dev-tools` 的 GitHub Releases API 逻辑描述

## 安装更新流程

Rust 侧命令：

```rust
install_update_and_restart(app: AppHandle) -> Result<bool, String>
```

大致流程：

1. 通过 updater endpoint 检查是否有可安装版本
2. 使用 `reqwest` 下载更新包到内存
3. 通过事件 `update-download-progress` 上报进度
4. 调用 `update.install(buffer)` 安装
5. Windows 下优先移除托盘图标，避免安装阶段残留托盘
6. 非 Windows 平台在安装后执行 `app.restart()`

## CI 发布

- Tag `v*` 触发 `.github/workflows/release.yml`
- 产物上传 GitHub Release + `latest.json`
- 更新说明：`release-notes/vX.Y.Z.md`
